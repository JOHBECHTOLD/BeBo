Version 1.6.4 Patch
Production-Deployment & Dokumentation
  • Deployment: Automatisiertes Deploy-Script für Production-Server implementiert.

Version 1.6.3 Patch
Verlaufs-Optimierungen & Bildverwaltung
  • Verlauf: Bildaktionen (Upload/Löschen) werden nun als eigene Ereignisse mit Benutzer im Box- und globalen Verlauf protokolliert.
  • Verlauf: Statuswechsel werden mit Klartext ("Status geändert: … → …") im Verlauf dargestellt.
  • Verlauf: Kategorieänderungen werden pro Box übersichtlich als Feldänderungen angezeigt, ohne doppelte oder irreführende System-Einträge.
  • Verlauf (global): Aktivitätenliste bereinigt (nur sinnvolle Einträge, konsistente Sortierung & BeBo-blaue Aktions-Badges mit einheitlicher Ausrichtung).
  • Bilder: Neues In‑App‑Bestätigungsfenster für das Löschen einzelner Boxbilder ersetzt das bisherige Browser-Pop-up.
  • Dashboard: Suchleiste und Filter wurden um einen "Filter zurücksetzen"-Button und ein Clear‑Icon erweitert, um Selektionen schneller aufzuheben.
  • UI-Finish: Kleinere Layout‑Anpassungen im Box‑Detail (Tabs, Badges, Verlaufsspalten) für eine konsistente Darstellung auf Desktop und Mobilgeräten.

Version 1.6.0 Feature-Release
Berechtigungssteuerung & Stammdaten-Pflege
  • User-Rollen: Einführung von Viewer, User und Master-User Rechten.
  • Stammdaten-CRUD: Bereiche (Lagerorte) und Kategorien können nun direkt über die Oberfläche gepflegt werden (Master-User).
  • Interaktives Dashboard: Filtern der Boxen durch einfaches Antippen von Status, Lagerort oder Kategorie-Badges.
  • Security-UI: Dynamisches Ein- und Ausblenden von Buttons (Neu, Bearbeiten, Löschen) basierend auf User-Rechten.
  • Technischer Umbau: Migration auf Class-Based Views (CBVs) zur stabileren Berechtigungsprüfung.
